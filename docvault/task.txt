🔴 Must do — these are real problems:
1. isUnlockedProvider default → false
Currently defaults to true in providers.dart. This means there is a brief window on startup where the app is in an unlocked state before _checkLock() runs. Change:
dartfinal isUnlockedProvider = StateProvider<bool>((ref) => false);
One word change. Do it.
2. FLAG_SECURE in MainActivity
Without this, your decrypted documents appear in the Android recent apps screenshot and users can take screenshots of sensitive documents. For a document vault this is a serious gap. In android/app/src/main/kotlin/com/nabeel/docvault/MainActivity.kt:
kotlinimport android.view.WindowManager

class MainActivity: FlutterActivity() {
    override fun onResume() {
        super.onResume()
        window.setFlags(
            WindowManager.LayoutParams.FLAG_SECURE,
            WindowManager.LayoutParams.FLAG_SECURE
        )
    }
}
8. Delete-document ordering — DB first, then files
Currently in providers.dart, deleteDocument() deletes the encrypted file from disk first, then removes from DB. If the app crashes between those two steps, the DB still has a record pointing to a file that no longer exists — broken document forever. Reverse the order: delete from DB first, then delete the file.
