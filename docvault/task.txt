[SUCCESS] Category icon input accepts any text, not just emoji
— Fixed: Added validation using `iconText.characters.length != 1`. This uses the 'characters' package to count visual grapheme clusters, strictly enforcing exactly one emoji per category and blocking normal text strings.

[SUCCESS] Enable deletion of categories (with "Other" protection)
— Fixed: Implemented `deleteCategory` in DatabaseService. It automatically reassigns documents from deleted categories to "Other". The "Other" category is strictly protected from deletion via a backend exception (`if (target.name == 'Other')`) and UI hiding.

[SUCCESS] Category Reordering
— Fixed: Implemented a custom `ReorderableListView` with `ClampingScrollPhysics` and `ReorderableDragStartListener` for smooth drag-and-drop. The exact order is saved to the database via a batch SQL update to the new `sort_order` column and instantly syncs across the app.

[SUCCESS] PIN UI Unification
— Fixed: Built a highly secure, custom `PinKeypad` widget. It has replaced the default system keyboard in both the Setup/Settings screen and the Lock screen. This prevents third-party keyboard keylogging and unifies the app's aesthetic.

[SUCCESS] No feedback when biometric auth fails silently
— Fixed: Updated `_tryBio()` in the Lock Screen. If the fingerprint or face scan fails, it now triggers a `ScaffoldMessenger` SnackBar that displays: "Biometric authentication failed, use your PIN.", eliminating the confusing silent failure.
