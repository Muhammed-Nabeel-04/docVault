Critical
resetOnError: true silently wipes encryption key on any secure storage error
lib/services/encryption_service.dart · lib/services/auth_service.dart
Both FlutterSecureStorage instances use resetOnError: true. On Android, if the Keystore has any issue (device restart, OS upgrade, biometrics changed), the entire keychain is silently deleted — including the AES key. All encrypted documents become permanently unreadable with no warning to the user. This is data loss, not a graceful recovery.
// Remove resetOnError or set it to false. Handle errors explicitly:
static const _storage = FlutterSecureStorage(
  aOptions: AndroidOptions(resetOnError: false),
);

    }
    // Android 13+: file_picker handles its own permissions
  }
  // ... rest of picker
}

Scheduled notifications fire at wrong time — tz.local defaults to UTC
lib/services/notification_service.dart · line 62
initializeTimeZones() is called but tz.local is never explicitly set to the device timezone. It defaults to UTC. For a user in India (UTC+5:30), a 9am expiry reminder fires at 2:30pm instead. The fix requires flutter_timezone or similar to read the device timezone and call tz.setLocalLocation().
import 'package:flutter_timezone/flutter_timezone.dart';

static Future<void> init() async {
  tz.initializeTimeZones();
  final tzName = await FlutterTimezone.getLocalTimezone();
  tz.setLocalLocation(tz.getLocation(tzName));
  // ...
}

  // Do NOT pop the screen on error — let user fix and retry

Category.toMap() includes id: null causing "UNIQUE constraint failed" on insert
lib/models/category.dart · toMap() / database_service.dart · addCategory()
Category.toMap() always includes the 'id' key even when id is null. SQLite with INTEGER PRIMARY KEY AUTOINCREMENT expects the key to be absent (not present as null) for auto-increment to work. Passing id: null explicitly can cause constraint errors in stricter SQLite builds.
Map<String, dynamic> toMap() {

    'name': name,
    'icon': icon,
  };
  if (id != null) map['id'] = id;
  return map;
}

PIN stored in plaintext — should be hashed
lib/services/auth_service.dart · setPin() / verifyPin()
The 4-digit PIN is stored directly in flutter_secure_storage and compared with == in plain text. If the secure storage is ever extracted (rooted device, backup), the PIN is immediately readable. A bcrypt or PBKDF2 hash with a salt should be stored instead.
